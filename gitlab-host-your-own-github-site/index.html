GitLab - Host your own GitHub site

I have a lot of git repos that I want to have centralized repos on since I also have a lot of servers that I need to push/fetch code to/from.

Github is great but I don’t want to keep all my repos public and I’d prefer to host them myself. I’ve been watching this project called GitLab for a while and last week I decided that it was time to try it.

TL;DR;
GitLab is great if you or your business partners want to have a central place for your git repos but you don’t want to be over at Github.

I imported a repo I currently work on and push all my work there, here are some screenshots from the UI. I haven’t found any major bugs yet.

Commits over time

Branch graphs

Browse old commits

Issues

The Issue list is quite simple but still very helpful. There are only two states of an issue, open or closed.

I can’t wait to test this with a repo that has many contributors to see if it’s as useful as it seems.
And of course, I host it inside a Docker.io container on one of my VPS:s. If I want to move it to another server, it’s dead simple.

So far I really like it!
